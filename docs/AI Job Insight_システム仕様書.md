# AI Job Insight システム仕様書

本書は AI Job Insight のシステム構成およびインフラ観点の仕様を定義する。

本書では主に以下を定義する。

- システム全体アーキテクチャ
- システム間の関係
- 技術スタック
- データ保存方式
- AIサービスとの通信方式

画面仕様や機能仕様などの詳細は、基本仕様書および詳細設計書で定義する。

---

# 1. システム概要

AI Job Insight は、クラウドワークス案件の応募価値分析および応募支援を行う Web アプリケーションである。

ユーザーが入力した

- 案件情報
- 自身のプロフィール

をもとに AI が分析を行い、応募判断や投稿文作成を支援する。

本システムはクライアントサイド中心の軽量な Web アプリとして設計する。

主な特徴
- 以下２つの通信方式に対応
- ユーザー自身がAI APIキーをブラウザに設定し、ブラウザから直接通信するフロントエンド通信方式に対応
- OAuth認証の場合に、バックエンドサーバーを介したAI API通信を行うバックエンド通信方式に対応

---

# 2. システム構成

## 2.1 全体アーキテクチャ

```
■v1 版 フロントエンド通信方式のみ
ユーザーブラウザ（Reactアプリ）
        ↓ HTTPS
外部AIサービス（OpenAI または Google Gemini API）

■v2版 バックエンド通信方式を追加 (Google OAuthへの対応に伴い追加実装する)
ユーザーブラウザ（Reactアプリ）
        ↓ HTTPS
サーバーバックエンド処理（Node.js）
        ↓ HTTPS
外部AIサービス（OpenAI または Google Gemini API）


```

v1における特徴

- 完全クライアント型 Webアプリ
- 静的ホスティングで運用可能
- サーバーサイド処理なし

v2における特徴

- クライアント/サーバー型の Webアプリ
- ログイン認証を使用する場合のみサーバーサイドで処理を行う
---

## 2.2 システム構成要素

本システムは以下のシステム要素で構成される。

### 1 本システム（フロントエンドアプリ）

ブラウザ上で動作する Web アプリケーション。

主な役割

- ユーザー入力の受付
- AI API 呼び出し
- 分析結果表示
- AIとの対話インターフェース
- 投稿文作成支援
- テンプレート管理
- 応募予定案件管理

技術

- React
- TypeScript

---

### 2 外部AIサービス

AIによる分析および文章生成を行う外部サービス。

利用するAIサービス

- OpenAI
- Google Gemini

主な用途

- 案件分析
- プロフィール文生成
- 投稿文作成支援
- AI対話支援
- テンプレート改善提案

通信方式

- HTTPS REST API

---

### 3 ブラウザストレージ

ユーザーが作成したデータをブラウザ内に保存する。

保存方式

- localStorage

将来拡張

- IndexedDB

保存データ例

- APIキー
- プロフィール情報
- 生成プロフィール文
- 案件履歴（案件内容/案件URL/案件メモ)
- 投稿文
- 投稿文テンプレート
- 応募予定案件リスト
- 各種設定

---

### 4 静的ホスティング環境

本システムのフロントエンドアプリを配信する環境。

候補

- Vercel
- Netlify
- Cloudflare Pages

必要条件

- HTTPS対応
- CDN配信

# 9. 設計方針まとめ

v1版は以下の方針で開発する。

1 クライアント中心アーキテクチャ

2 バックエンドなし(v2以降で追加)

3 データベースなし(v2以降で必要に応じて追加)

4 静的Webアプリ

5 ユーザーAPIキー方式

この構成により

- 開発スピード
- 運用コスト削減
- シンプルなシステム構造

を優先する。
