# AI Job Insight 詳細設計書 - AI応答サービス機能

---

# 1. 機能概要

## 1.1 機能名

AI応答サービス機能

## 1.2 機能ID

FUNC-AI-SERVICE-001

## 1.3 機能の目的

外部AIサービスとの通信を行い、分析リクエストの送信およびレスポンスの受信処理を実行する。取得したAI応答を、アプリケーション内で利用可能な形式へ整形する。

## 1.4 利用される画面

- なし（データ処理機能のため画面なし。v1ではフロントエンド実装、v2以降はバックエンド実装も追加）

---

# 2. 処理の実現方式

この機能は以下の技術・方式で実現する。

- 言語：TypeScript
- 通信方式：HTTPS REST API
- 非同期処理：async / await
- エラーハンドリング：try / catch
- v1：フロントエンド側に実装
- v2以降：フロントエンド側とバックエンド側それぞれに実装

対応AIサービス

- OpenAI API
- Google Gemini API

v2以降の追加対応

- Google Gemini API（OAuth認証によるバックエンド通信）

---

# 3. 入力データ

|項目名|型|必須|説明|
|---|---|---|---|
|AIサービス種別|string|○|使用するAIサービス（OpenAI / Gemini）|
|APIキー|string|○（v1）|v1で使用するAPIキー|
|プロンプト|string|○|AIへ送信するプロンプト文|
|モデル名|string|○|使用するAIモデル名|
|温度パラメータ|number|任意|生成の多様性を制御するパラメータ（デフォルト：0.7）|

---

# 4. 出力データ

|項目名|型|説明|
|---|---|---|
|生レスポンス|string|AIから返却された生の応答テキスト|
|パース済みデータ|object|JSON形式にパースされた応答データ|
|エラー情報|object|エラー発生時の詳細情報|

---

# 5. 処理の流れ

## 5.1 AI通信処理ブロック

1. AIサービス種別の判定
2. 該当するAI APIのエンドポイントURLを取得
3. リクエストヘッダーの構築
4. リクエストボディの構築
5. AI APIへHTTPSリクエスト送信
6. レスポンス待機
7. レスポンス受信
8. HTTPステータスコードの確認
9. エラーがある場合はエラー情報を返却
10. 正常な場合はレスポンスボディを返却

## 5.2 AI応答整形ブロック

1. 生レスポンステキストの取得
2. JSON形式への変換試行
3. パース成功時は構造化データとして返却
4. パース失敗時はテキストとして返却
5. 必要に応じてデータ構造の正規化
6. 画面表示用フォーマットへの変換
7. 履歴保存用フォーマットへの変換
8. 整形済みデータの返却

# 6. 使用するデータ

|データ名|保存場所|説明|
|---|---|---|
|APIキー|localStorage（v1）|ユーザーが設定したAPIキー|
|OAuth認証情報|バックエンドセッション（v2以降）|OAuth認証で取得した認証情報|
|AIサービス設定|localStorage|使用するAIサービスの種別情報|

---

# 7. AIプロンプト仕様

この機能ではプロンプトの内容は定義せず、呼び出し元から受け取ったプロンプトをそのままAI APIへ送信する。

プロンプトの内容定義は各呼び出し元機能で行う。

---

# 8. エラー処理

|エラー内容|対応|
|---|---|
|APIキー未設定|エラーコード「API_KEY_MISSING」を返却|
|ネットワークエラー|エラーコード「NETWORK_ERROR」を返却|
|HTTP 401エラー|エラーコード「UNAUTHORIZED」を返却（APIキー無効）|
|HTTP 429エラー|エラーコード「RATE_LIMIT」を返却（レート制限）|
|HTTP 500エラー|エラーコード「SERVER_ERROR」を返却（サーバーエラー）|
|タイムアウト|エラーコード「TIMEOUT」を返却|
|レスポンスパースエラー|エラーコード「PARSE_ERROR」を返却|
|不明なエラー|エラーコード「UNKNOWN_ERROR」を返却|

# 10. 制約事項

- 通信はHTTPSのみ対応
- タイムアウト時間：30秒
- リトライ回数：なし（エラー時はユーザーが再実行）
- レート制限はAIサービス側の仕様に依存
- v1ではCORS制約に注意が必要
- v2以降のOAuth認証はバックエンド側で処理

# 12. 備考

## OpenAI API仕様

- エンドポイント：https://api.openai.com/v1/chat/completions
- 認証方式：Bearer Token
- リクエスト形式：JSON
- レスポンス形式：JSON

## Google Gemini API仕様（v1）

- エンドポイント：https://generativelanguage.googleapis.com/v1/models/{model}:generateContent
- 認証方式：APIキーパラメータ
- リクエスト形式：JSON
- レスポンス形式：JSON

## Google Gemini API仕様（v2以降 OAuth）

- OAuth認証経由での利用
- バックエンドサーバー経由で通信
- ※本機能はv2以降で実装予定

## エラーコード一覧

- API_KEY_MISSING：APIキー未設定
- NETWORK_ERROR：ネットワークエラー
- UNAUTHORIZED：認証エラー
- RATE_LIMIT：レート制限エラー
- SERVER_ERROR：サーバーエラー
- TIMEOUT：タイムアウトエラー
- PARSE_ERROR：パースエラー
- UNKNOWN_ERROR：不明なエラー
