# AI Job Insight 詳細設計書 - 入力管理機能

---

# 1. 機能概要

## 1.1 機能名

入力管理機能

## 1.2 機能ID

FUNC-INPUT-001

## 1.3 機能の目的

ユーザーが案件情報およびプロフィール情報を入力・保存・管理する。APIキーの設定、プロフィール文の生成、入力データの検証を行う。

## 1.4 利用される画面

- メイン画面 APIキー設定領域
- メイン画面 プロフィール入力領域
- メイン画面 案件入力領域

---

# 2. 処理の実現方式

この機能は以下の技術・方式で実現する。

- フロントエンド：React
- 言語：TypeScript
- データ保存：localStorage
- データ形式：JSON
- 入力検証：バリデーション処理

---

# 3. 入力データ

## 3.1 APIキー設定

|項目名|型|必須|説明|
|---|---|---|---|
|AIサービス種別|string|○|使用するAIサービス（OpenAI / Gemini）|
|APIキー|string|任意|AIサービスのAPIキー（空の場合はデモモード）|
|モデル名|string|任意|使用するAIモデル名（候補から選択または直接入力）|

**モデル名入力の仕様（v1.1以降）**
- input + datalist 方式のハイブリッドUI
- 候補リストから選択または直接入力が可能
- サービス切替時に自動でデフォルトモデルを設定
- 各サービスのモデル候補：
  - **OpenAI**: gpt-4o-mini（推奨）、gpt-4o、gpt-4-turbo、gpt-4、o1-mini、o1
  - **Gemini**: gemini-1.5-flash（推奨）、gemini-1.5-pro、gemini-2.0-flash-exp、gemini-exp-1206
- デフォルトモデル：
  - **OpenAI**: gpt-4o-mini
  - **Gemini**: gemini-1.5-flash
- モデル一覧の公式ドキュメントリンクを表示

## 3.2 プロフィール入力

|項目名|型|必須|説明|
|---|---|---|---|
|自己紹介|string|○|プロフィールの自己紹介|
|スキル|array[string]|○|プロフィールのスキル情報（タグ形式で入力・表示）|
|実績|string|○|プロフィールの実績情報|
|得意分野|string|○|プロフィールの得意分野|
|プロフィール文文字数上限|number|任意|生成するプロフィール文の文字数上限（デフォルト1000）|

**スキル入力の仕様**
- タグUI形式で入力・表示
- Enterキーで確定して複数のスキルタグを追加
- カンマ区切りで複数スキルを一括入力可能（例：「React,Vue,Angular」→3つのタグとして追加）
- 各スキルは前後の空白を自動削除
- 重複スキルは自動的に排除
- 各タグに削除ボタン（×）を表示
- レーダーチャートの軸ラベルとして使用

## 3.3 案件入力

|項目名|型|必須|説明|
|---|---|---|---|
|案件説明|string|○|案件説明文（仕事内容、条件、報酬を含む全文をコピー&ペースト）|
|案件URL|string|任意|案件URL|
|案件メモ|string|任意|ユーザーのメモ|

**案件説明の想定内容**
- クラウドソーシングサイトの案件詳細をそのままコピー&ペースト
- 仕事内容、必須スキル、条件、報酬などがすべて含まれる
- AIが内容を解析して分析に使用

|項目名|型|説明|
|---|---|---|
|保存完了フラグ|boolean|保存成功／失敗|
|エラーメッセージ|string|エラー発生時のメッセージ|
|生成プロフィール文|string|AIが生成したプロフィール文|

# 5. 処理の流れ

## 5.1 APIキー設定

1. ユーザーがAIサービス種別を選択
   - サービス切替時に自動でデフォルトモデルを設定
2. ユーザーがAPIキーを入力（任意）
3. ユーザーがモデル名を入力（任意）
   - input + datalist 方式で候補から選択または直接入力
   - プレースホルダーにデフォルトモデルを表示
   - モデル一覧の公式ドキュメントリンクを表示
4. 保存ボタンを押下
5. 入力検証を実施
   - APIキーとモデル名が両方空の場合：デモモードとして保存
   - APIキーとモデル名が両方入力されている場合：通常モードとして保存
   - どちらか一方のみ入力されている場合：エラー
6. 検証成功時
   - localStorageへ保存
   - 成功メッセージを表示
7. 検証失敗時
   - エラーメッセージを表示

## 5.2 プロフィール入力

1. ユーザーがプロフィール情報を入力
2. 入力内容を一時保持
3. フォーカスアウト時または一定時間経過後に自動保存
4. localStorageへ保存
5. スキル項目ラベルが入力された場合
   - カンマ区切りで配列に変換
   - localStorageへ保存
6. プロフィール文文字数上限が変更された場合
   - 数値検証を実施
   - localStorageへ保存

## 5.3 プロフィール文生成

1. ユーザーがプロフィール文作成ボタンを押下
2. 入力検証を実施
   - 必須項目が入力されているか
3. 検証成功時
   - AI応答サービス機能へリクエスト送信
   - ローディング表示開始
   - AI生成プロフィール文を受信
   - AI提案として表示
   - ユーザーが承認ボタンを押下
   - localStorageへ保存
   - 生成プロフィール文表示エリアへ表示
   - ローディング表示終了
4. 検証失敗時
   - エラーメッセージを表示

## 5.4 案件入力

1. ユーザーが案件情報を入力
2. 入力内容を一時保持
3. 分析実行時に入力検証を実施
   - 必須項目が入力されているか
4. 検証成功時
   - 分析実行機能へデータを渡す
5. 検証失敗時
   - エラーメッセージを表示

## 5.5 入力データ復元

1. アプリ起動時にlocalStorageからデータを取得
2. APIキー設定を復元
3. プロフィール情報を復元
4. 生成プロフィール文を復元
5. 各入力フィールドへ値を設定

# 8. エラー処理

|エラー内容|対応|
|---|---|
|APIキー未入力|エラーメッセージ「APIキーを入力してください」を表示|
|APIキー形式エラー|エラーメッセージ「APIキーの形式が正しくありません」を表示|
|モデル名未入力|エラーメッセージ「モデル名を入力してください」を表示|
|プロフィール必須項目未入力|エラーメッセージ「プロフィール情報をすべて入力してください」を表示|
|案件必須項目未入力|エラーメッセージ「案件情報をすべて入力してください」を表示|
|文字数上限入力エラー|エラーメッセージ「文字数上限は1以上の数値を入力してください」を表示|
|プロフィール文生成失敗|エラーメッセージ「プロフィール文の生成に失敗しました」を表示|
|データ保存失敗|エラーメッセージ「データの保存に失敗しました」を表示|

# 12. 備考

## APIキー入力フィールド仕様

- 入力形式：パスワード（マスク表示）
- プレースホルダー：「sk-...」（OpenAI）または「AI...」（Gemini）
- モデル名入力フィールド：テキスト（例：gpt-4, gemini-pro）
- 保存ボタン：入力後に活性化

## プロフィール入力フィールド仕様

- 自己紹介：複数行テキスト（rows: 5）
- スキル：複数行テキスト（rows: 5）
- 実績：複数行テキスト（rows: 5）
- 得意分野：複数行テキスト（rows: 3）
- スキル項目ラベル：単一行テキスト（カンマ区切り）
- プロフィール文文字数上限：数値入力（デフォルト: 1000）

## 案件入力フィールド仕様

- 案件説明：複数行テキスト（rows: 10）
- 案件URL：単一行テキスト（URL形式）
- 案件メモ：複数行テキスト（rows: 3）

## 生成プロフィール文表示エリア仕様

- 表示形式：読み取り専用テキストエリア
- コピーボタン：クリップボードへコピー
- 文字数表示：現在の文字数を表示

## 入力検証ルール

### APIキー

- 空でないこと
- 最小文字数：10文字以上

### モデル名

- 空でないこと
- 最小文字数：3文字以上

### プロフィール必須項目

- 空でないこと
- 最小文字数：10文字以上

### 案件必須項目

- 空でないこと
- 最小文字数：10文字以上

### プロフィール文文字数上限

- 1以上の整数
- 最大値：10000

## データ保存タイミング

- APIキー：保存ボタン押下時
- モデル名：保存ボタン押下時
- プロフィール情報：フォーカスアウト時または5秒経過後
- 生成プロフィール文：AI提案承認時
- 案件情報：一時保持のみ（分析実行時に使用）

## AI提案承認方式

プロフィール文生成時には、AIが生成した内容を提案として表示し、ユーザーが承認した場合にのみ保存する。

承認フロー

1. AI生成プロフィール文を提案表示
2. ユーザーが内容を確認
3. 承認ボタンを押下
4. localStorageへ保存
5. 生成プロフィール文表示エリアへ反映

# 12. デモモード時の動作

## 12.1 初期データ自動表示

デモモード時（APIキー未設定時）は、プロフィールと案件の入力欄に自動的にデモデータが表示される。

### プロフィールデモデータ

- **自己紹介**: Webライターのサンプルプロフィール
- **スキル**: ['SEO記事作成', 'WordPress入稿', 'リサーチ', 'データ分析', 'ChatGPT活用']
- **実績**: IT系メディア執筆実績
- **得意分野**: AI / IT / SaaS等

### 案件デモデータ

- **案件説明**: SEO記事執筆案件のサンプル
- **案件URL**: サンプルURL
- **案件メモ**: 継続案件の可能性についてのメモ

## 12.2 入力クリア動作

ユーザーが明示的にデータを削除した場合（data === null）は、デモモードかどうかに関わらず入力欄をクリアする。

### クリア条件

```typescript
if (data === null) {
  // デモモードでも通常モードでもクリア
  clearAllFields();
}
```

## 12.3 データ永続化

デモモード時の入力データは、専用のストレージキー（`_demo` サフィックス）で保存される。通常モードのデータとは分離される。
