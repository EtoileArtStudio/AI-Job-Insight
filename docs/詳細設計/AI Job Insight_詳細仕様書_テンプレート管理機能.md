# AI Job Insight 詳細設計書 - テンプレート管理機能

---

# 1. 機能概要

## 1.1 機能名

テンプレート管理機能

## 1.2 機能ID

FUNC-TEMPLATE-001

## 1.3 機能の目的

作成した投稿文を再利用可能なテンプレートとして保存・編集・呼び出し・削除する。AIとの対話によりテンプレートをブラッシュアップし、提案内容をユーザー承認後に反映する。

## 1.4 利用される画面

- メイン画面 テンプレート一覧・確認領域
- メイン画面 対話支援領域

---

# 2. 処理の実現方式

この機能は以下の技術・方式で実現する。

- フロントエンド：React
- 言語：TypeScript
- データ保存：localStorage
- AI呼び出し：AI応答サービス機能を利用
- データ形式：JSON

---

# 3. 入力データ

|項目名|型|必須|説明|
|---|---|---|---|
|投稿文|string|○|テンプレート化する投稿文|
|テンプレート名|string|○|テンプレートの識別名|
|テンプレートID|string|○（編集時）|編集対象のテンプレートID|
|AI整形フラグ|boolean|○|AI整形を使用するかどうか|

---

# 4. 出力データ

|項目名|型|説明|
|---|---|---|
|テンプレート一覧|array|保存済みテンプレートの配列|
|テンプレート内容|object|選択したテンプレートの詳細|
|AI提案内容|string|AIが提案した改善内容|

---

# 5. 処理の流れ

## 5.1 テンプレート保存

1. ユーザーが投稿文作成モードで投稿文を作成
2. テンプレート化ボタンを押下
3. テンプレート保存方式を選択
   - AIによる成形処理を行う
   - 手作業で編集した内容をそのまま保存
4. AI整形を選択した場合
   - AI応答サービス機能へテンプレート整形リクエスト送信
   - AI提案内容を表示
   - ユーザーが承認ボタンを押下
   - テンプレートとして保存
5. 手作業を選択した場合
   - そのままテンプレートとして保存
6. テンプレート名を入力
7. localStorageへ保存
8. テンプレート一覧を更新

## 5.2 テンプレート一覧表示

1. localStorageからテンプレートデータを取得
2. テンプレート一覧をレンダリング
3. テンプレート名、作成日時を表示

## 5.3 テンプレート内容確認

1. ユーザーがテンプレート内容確認ボタンを押下
2. テンプレートIDを取得
3. localStorageから該当テンプレートを取得
4. テンプレート内容を表示

## 5.4 テンプレート手動編集

1. ユーザーがテンプレート編集ボタンを押下
2. テンプレートIDを取得
3. localStorageから該当テンプレートを取得
4. 編集画面を表示
5. ユーザーが内容を編集
6. 保存ボタンを押下
7. localStorageへ上書き保存
8. テンプレート一覧を更新

## 5.5 テンプレートブラッシュアップ

1. ユーザーがテンプレートブラッシュアップボタンを押下
2. テンプレートIDを取得
3. localStorageから該当テンプレートを取得
4. AIとの対話インターフェースを表示
5. ユーザーがAIへ改善依頼を入力
6. AI応答サービス機能へリクエスト送信
7. AI提案内容を表示
8. ユーザーが承認ボタンを押下
9. テンプレート本体へ反映
10. localStorageへ上書き保存
11. テンプレート一覧を更新

## 5.6 テンプレート削除

1. ユーザーがテンプレート削除ボタンを押下
2. 確認ダイアログを表示
3. ユーザーが削除を確認
4. テンプレートIDを取得
5. localStorageから該当テンプレートを削除
6. テンプレート一覧を更新

## 5.7 テンプレート読込

1. ユーザーが投稿文作成モードで既存テンプレート使用を選択
2. テンプレート選択ドロップダウンを表示
3. ユーザーがテンプレートを選択
4. テンプレートIDを取得
5. localStorageから該当テンプレートを取得
6. 投稿文作成エリアへテンプレート内容を表示
7. AIとの対話で内容を調整

---

# 6. 使用するデータ

|データ名|保存場所|説明|
|---|---|---|
|テンプレートデータ|localStorage|保存済みテンプレートの配列|
|未承認AI提案データ|一時保持|AIが提示した未承認の改善提案|

---

# 7. AIプロンプト仕様

## 7.1 テンプレート整形時のプロンプト

```
以下の投稿文を、再利用可能なテンプレートとして整形してください。

投稿文：
{投稿文}

整形の方針：
- 案件固有の情報は{案件名}、{報酬}などのプレースホルダーに置き換える
- 汎用的に使える表現にする
- 文章構成は維持する

整形後のテンプレートを出力してください。
```

## 7.2 テンプレートブラッシュアップ時のプロンプト

```
以下のテンプレートを改善してください。

現在のテンプレート：
{テンプレート内容}

改善依頼：
{ユーザーからの改善依頼}

改善後のテンプレートを出力してください。
```

# 8. エラー処理

|エラー内容|対応|
|---|---|
|テンプレート名未入力|エラーメッセージ「テンプレート名を入力してください」を表示|
|テンプレート保存失敗|エラーメッセージ「テンプレートの保存に失敗しました」を表示|
|テンプレート取得失敗|エラーメッセージ「テンプレートの取得に失敗しました」を表示|
|AI整形失敗|エラーメッセージ「AI整形に失敗しました」を表示し、手作業保存を提案|
|テンプレート削除失敗|エラーメッセージ「テンプレートの削除に失敗しました」を表示|

# 10. 制約事項

- テンプレート数の上限は設けない（localStorageの容量制限に依存）
- テンプレートIDはUUID形式
- AI提案内容は承認されるまでテンプレート本体を書き換えない
- 削除したテンプレートは復元できない

## AI提案承認方式

AIが生成または改善提案した結果は、以下の手順で反映する。

1. AI提案を表示
2. ユーザー確認
3. 承認ボタン押下
4. データ反映

AIは保存データを自動で書き換えない。

## テンプレート保存方式

ユーザーは以下の方法を選択できる。

- AIによる成形処理を行ってテンプレート化する
- ユーザーが手作業で編集した内容をそのままテンプレート化する

## プレースホルダー例

- {案件名}
- {報酬}
- {納期}
- {スキル}
- {実績}
