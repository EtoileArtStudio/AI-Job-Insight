# AI Job Insight 詳細設計書テンプレート

本ドキュメントは AI Job Insight の各機能の詳細設計を記述するためのテンプレートである。

---

# 記載ルール

詳細設計書を作成する際は、以下のルールに従って記述する。

1. **該当しない項目は必ず「なし」と記載する**  
   例：外部APIを使用しない場合 → 「使用外部API：なし」

2. **空欄は作らない**  
   情報が存在しないのか、未記入なのかが区別できなくなるため。

3. **将来機能は明確に区別する**  
   v2以降の機能の場合は
   「※本機能はv2以降で実装予定」
   と記載する。

4. **設計対象外の場合は理由を書く**  
   例：
   「本機能ではデータ保存を行わないため該当なし」

5. **データ項目が存在しない場合**

|項目|記載方法|
|---|---|
|入力データ|なし|
|出力データ|なし|
|保存データ|なし|

6. **エラー処理が存在しない場合**

|エラー内容|対応|
|---|---|
|該当なし|本機能ではエラー処理を実装しない|

7. **関連機能が存在しない場合**

```
関連機能：なし
```

8. **AI機能を使用しない機能の場合**

```
AIプロンプト仕様：なし
```

9. **仕様変更を考慮し、文章は断定形で記述する**

例

良い例

「案件テキストを取得する」

悪い例

「案件テキストを取得することができる」

この機能をどのような技術・方式で実現するかを記述する。

例：

- フロントエンド：React

---

# 3. 入力データ

|項目名|型|必須|説明|
|---|---|---|---|
|案件内容|string|○|ユーザーが入力した案件説明|
|プロフィール|string|○|ユーザーの自己紹介文|
|テンプレート|string|任意|提案文テンプレート|

---

# 4. 出力データ

|項目名|型|説明|
|---|---|---|
|おすすめ度|number|1〜5の評価|
|分析コメント|string|AIによる評価|
|応募戦略|string|応募のためのアドバイス|

---

# 5. 処理の流れ

1. ユーザーが分析実行ボタンを押す
2. 入力データを取得
3. 入力チェックを実施
4. AI APIへリクエスト送信
5. AIレスポンス取得
6. レスポンス整形
7. 結果表示機能へデータを渡す

---

# 6. 使用するデータ

|データ名|保存場所|説明|
|---|---|---|
|分析履歴|LocalStorage|過去の分析結果|
|テンプレート|LocalStorage|ユーザー作成テンプレート|

---

# 7. AIプロンプト仕様

AIに送信するプロンプト構造を定義する。

## 7.1 プロンプト構造

例：

```
あなたはクラウドソーシング案件分析AIです。

以下の案件について分析してください。

案件内容：
{job_description}

プロフィール：
{profile}

次の形式で回答してください。

おすすめ度：1〜5
理由：
応募戦略：
```

## 7.2 出力フォーマット

例：

JSON形式

```
{
  "score": 4,
  "reason": "スキルとの一致度が高い",
  "strategy": "過去の実績を強調すると良い"
}
```
