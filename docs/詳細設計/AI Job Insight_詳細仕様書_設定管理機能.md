# AI Job Insight 詳細設計書 - 設定管理機能

---

# 1. 機能概要

## 1.1 機能名

設定管理機能

## 1.2 機能ID

FUNC-SETTINGS-001

## 1.3 機能の目的

アプリ内で保持している設定値および保存データをユーザーが管理する。各種データの削除、設定値のリセット、全データの初期化を行う。

## 1.4 利用される画面

- 設定画面

---

# 2. 処理の実現方式

この機能は以下の技術・方式で実現する。

- フロントエンド：React
- 言語：TypeScript
- データ保存：localStorage
- データ形式：JSON

## 設定画面レイアウト

設定画面は**モーダル形式**で表示される。

### モーダル全体デザイン

- 表示方式：オーバーレイモーダル（画面中央に表示）
- 背景オーバーレイ：半透明の黒（rgba(0,0,0,0.5)）
- モーダルサイズ：固定幅（約600-700px）、高さは可変（スクロール可能）
- 背景色：白（#FFFFFF）
- 角丸：大きめ（border-radius: 12px）
- シャドウ：濃いめの影（box-shadow: 0 10px 30px rgba(0,0,0,0.2)）
- パディング：内側に十分な余白（24-32px）

### ヘッダー部分

- 画面タイトル：「設定」または「Settings」
- フォントサイズ：大きめ（24px）
- フォントウェイト：太字（font-weight: 600）
- 配置：左寄せ
- 閉じるボタン（×印）：
  - 配置：右上
  - アイコン色：グレー（#6B7280）
  - ホバー時：濃いグレーに変化
  - サイズ：24px × 24px
  - 機能：モーダルを閉じてメイン画面に戻る

### v1での構成

#### データ管理セクション

個別削除メニュー：

1. **プロフィールデータ削除**
   - プロフィール情報（`PROFILE_DATA`）
   - 生成されたプロフィール文（`GENERATED_PROFILE`）

2. **案件データ削除**
   - 案件情報（`JOB_DATA`）
   - 分析結果（`ANALYSIS_RESULT`）
   - 分析履歴（`ANALYSIS_HISTORY`）
   - **注**: 案件を削除すると、その案件に関連する分析結果も一緒に削除される

3. **応募文データ削除**
   - 応募文（`APPLICATION_TEXT`）
   - 案件ごとの応募文（`APPLICATION_DRAFTS`）
   - 応募文チャット履歴（`APPLICATION_CHAT_HISTORIES`）
   - 汎用応募文（`APPLICATION_TEXT_GENERIC`）
   - 汎用チャット履歴（`APPLICATION_GENERIC_CHAT`）
   - 分析チャット履歴（`ANALYSIS_CHAT_HISTORIES`）

削除処理の実装：
- 各削除処理後、`window.location.reload()`でページをリロード
- `useLocalStorage`フックが最新の値を再読込
- デモモード時は、削除後もデモデータが表示される

#### 全データ削除セクション

- セクションタイトル「すべてのデータを削除」
- 警告メッセージ：「APIキーは削除されません」
- 全データ削除ボタン
  - プロフィール、案件データ、分析結果、応募文をすべて削除
  - APIキー設定は保持

## 全保存データ初期化の実装詳細

全保存データ初期化は、アプリが管理する特定のlocalStorageキーのみを削除する。

### キー名前空間

- すべてのアプリ管理データは `aijobinsight_` プレフィックスを使用
- 初期化時は `aijobinsight_` で始まるキーのみを削除
- 同一オリジンの他アプリケーションデータは削除しない

### 削除対象キーの例

- `aijobinsight_apikey`
- `aijobinsight_profile`
- `aijobinsight_templates`
- `aijobinsight_history`
- など

※本機能はv2以降でログイン認証機能と連携

# 12. デモモード対応

## 12.1 概要

APIキーまたはモデル名が未設定の場合、デモモードが自動的に有効化される。デモモード時は専用のストレージキーを使用してデータを保存する。

## 12.2 ストレージキー分離

デモモードと通常モードでデータを分離するため、`getContextualStorageKey()`関数を使用する。

```typescript
function getContextualStorageKey(key: string): string {
  // API設定は常に通常キーを使用（デモモード判定の基準となるため）
  if (key === STORAGE_KEYS.API_KEY_CONFIG) {
    return key;
  }

  if (isDemoMode()) {
    // すべてのキーに"_demo"サフィックスを追加
    return `${key}_demo`;
  }
  return key;
}
```

### API設定キーの例外処理

`API_KEY_CONFIG`は常に通常キーで保存される。これは：
- デモモード判定の基準となるため
- APIキー設定後、即座に通常モードへ切り替える必要があるため

### デモモード時のキー例

- `aijobinsight_profile_data_demo`
- `aijobinsight_job_data_demo`
- `aijobinsight_analysis_history_demo`
- `aijobinsight_analysis_result_demo`

### モード切替時のデータ再読込

`useLocalStorage`フックは、`contextualKey`の変更を`useEffect`で監視し、モードが切り替わった際に自動的に適切なストレージからデータを再読込する。

## 12.3 デモモードからの移行

APIキーを設定すると、デモモードから通常モードに自動的に切り替わる。

### 移行時の動作

1. デモモードデータは保持される（`_demo` サフィックス付きキー）
2. 通常モードでは新しいストレージキーが使用される
3. ユーザーは手動でデモデータをクリアする必要がある

## 12.4 デモモードバナー

画面上部に固定表示されるバナーで、デモモード中であることをユーザーに通知する。

### バナーの内容

- 表示位置：画面最上部（固定）
- 背景色：警告色（黄色系）
- メッセージ：「デモモード - APIキーを設定すると全機能が利用できます」
- ツールチップ：デモモードの説明と制限事項

## 12.5 APIキー設定画面でのデモモード表示

APIキーが未設定の場合、注意メッセージを表示する。

### メッセージ内容

> ⚠️ デモモード中
> 
> 現在、APIキーが設定されていないためデモモードで動作しています。
> デモモードでは固定のサンプルデータが表示されます。
> 
> 実際のAI分析を利用するには、APIキーを設定してください。
