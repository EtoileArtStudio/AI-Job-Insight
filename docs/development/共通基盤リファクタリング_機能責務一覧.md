## 1.1 共通AI応答基盤機能

### 1.1.1 AI通信処理
**責務**
- AIサービスとのHTTPS通信
- エンドポイントURL管理
- リクエストヘッダー/ボディ構築
- HTTPステータスコード確認
- タイムアウト管理
- エラーハンドリング

**対象AIサービス**
- OpenAI API
- Google Gemini API

**実装場所（想定）**
- `src/core/ai/client.ts`
- `src/core/ai/config.ts`

**入力パラメータ（汎用）**
- service: 'openai' | 'gemini'
- apiKey: string
- prompt: string
- modelName: string
- temperature?: number

**出力（汎用）**
- rawResponse: string
- error?: Error

---

### 1.1.2 AI応答整形処理
**責務**
- 生レスポンステキストの正規化
- JSON形式への変換
- パースエラーハンドリング
- 構造化データへの変換
- 汎用データ形式への変換

**実装場所（想定）**
- `src/core/ai/transform.ts`

**入力**
- rawResponse: string
- responseFormat?: 'json' | 'text'

**出力（汎用構造）**
```typescript
interface GenericAIResponse {
  content: string;
  structuredData?: Record<string, any>;
  parseError?: string;
}
```

# 3. 責務境界の整理方針

## 3.1 基本原則

1. **共通機能は意味づけを持たない**
   - 「案件」「プロフィール」などのJob固有用語を含まない
   - 「分析」「評価」「スキル」などの汎用的な概念のみ扱う

2. **Job固有機能は共通機能を組み合わせて利用**
   - 共通AIクライアントを使ってJob用プロンプトを送信
   - 共通可視化コンポーネントにJob用データを渡して表示

3. **データ構造は汎用的に設計**
   - 共通機能で扱うデータ構造は拡張可能な設計
   - Job固有項目は`metadata`や拡張プロパティとして保持

## 3.2 依存方向

```
Job固有機能 → 共通機能
（依存する）  （依存されない）
```

共通機能はJob固有機能に依存しない。
Job固有機能は共通機能を利用する。
