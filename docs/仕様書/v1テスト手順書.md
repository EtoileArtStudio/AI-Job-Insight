# v1 実装テスト手順

## 開発環境の確認

### 開発サーバーの起動確認
```bash
npm run dev
```

ブラウザで http://localhost:5174 (または表示されたポート) にアクセス

## 機能テスト

### 1. API設定のテスト

1. ブラウザでアプリケーションを開く
2. サイドバーの「API設定」をクリック
3. 以下を入力:
   - サービス: OpenAI を選択
   - APIキー: `sk-test-xxxxx` (テスト用の値)
   - モデル名: `gpt-4`
4. 「保存」ボタンをクリック
5. ページをリロードして、設定が保存されているか確認

**期待結果**: 
- 入力値が保存される
- リロード後も設定が保持される
- localStorage に `aijobinsight_apiConfig` キーでデータが保存される

### 2. プロフィール入力のテスト

1. サイドバーの「プロフィール入力」をクリック
2. 以下を入力:
   - 自己紹介: 「Webエンジニア歴5年」
   - スキル: 「React, TypeScript, Node.js」
   - 実績: 「大規模ECサイトの開発」
   - 得意分野: 「フロントエンド開発」
   - スキルラベル: 「React, TypeScript」
3. 自動保存を確認（保存ボタンなしで保存される）
4. ページをリロードして、入力内容が保持されているか確認

**期待結果**:
- 入力内容が自動保存される
- リロード後も入力内容が保持される
- localStorage に `aijobinsight_profile` キーでデータが保存される

### 3. 案件入力のテスト

1. サイドバーの「案件入力」をクリック
2. 以下を入力:
   - 案件説明: 「ReactでのWebアプリ開発」
   - 業務内容: 「管理画面の実装」
   - 必須要件: 「React経験2年以上」
   - 報酬: 「月額50万円」
   - 案件URL: 「https://example.com/job/123」
   - メモ: 「興味あり」
3. 自動保存を確認
4. ページをリロードして、入力内容が保持されているか確認

**期待結果**:
- 入力内容が自動保存される
- リロード後も入力内容が保持される
- localStorage に `aijobinsight_job` キーでデータが保存される

### 4. 分析実行のテスト（要実APIキー）

**注意**: この機能は実際のAPIキーが必要です

1. 有効なAPIキーを設定（OpenAI または Gemini）
2. プロフィールと案件情報を入力
3. サイドバーの「分析実行」をクリック
4. 「分析実行」ボタンをクリック
5. ローディング状態が表示されることを確認
6. 分析結果が表示されることを確認:
   - 適合度（星1〜5）
   - 強み（緑色のチェックマーク付きリスト）
   - 懸念点（黄色の警告アイコン付きリスト）
   - スキルマッチ度（プログレスバー）
   - 応募戦略（青色の背景）

**期待結果**:
- API呼び出しが成功する
- 分析結果が適切に表示される
- localStorage に `aijobinsight_analysisResult` キーでデータが保存される

### 5. チャット機能のテスト（要実APIキー）

**前提条件**: 分析結果が表示されている状態

1. 右側のチャットエリアにメッセージを入力
   例: 「この案件の報酬は適正ですか？」
2. 送信ボタン（➤）をクリック
3. ユーザーメッセージが青色の吹き出しで右側に表示されることを確認
4. AIの応答が白色の吹き出しで左側に表示されることを確認
5. Enterキーでも送信できることを確認

**期待結果**:
- メッセージが送信される
- AIから応答が返ってくる
- メッセージの履歴が表示される
- プロフィール、案件、分析結果の情報を元に回答される

### 6. 設定モーダルのテスト

1. サイドバーの設定アイコン（⚙️）をクリック
2. モーダルが表示されることを確認
3. 「プロフィールデータ」の削除ボタンをクリック
4. 確認ダイアログで「OK」をクリック
5. プロフィールデータが削除されることを確認
6. プロフィール入力画面に移動して、データがクリアされていることを確認

**期待結果**:
- モーダルが正しく表示される
- 削除確認ダイアログが表示される
- データが削除される
- 削除後、該当データがクリアされる

### 7. 全データ削除のテスト

1. 設定モーダルを開く
2. 「すべてのデータを削除」ボタンをクリック
3. 確認ダイアログで「OK」をクリック
4. すべてのデータが削除されることを確認
5. プロフィール、案件、分析結果がすべてクリアされることを確認

**期待結果**:
- すべてのデータが削除される
- APIキーは削除されない

## エラーハンドリング

### API設定なしで分析実行

1. API設定をクリアする（設定モーダルは削除しないので手動でlocalStorageから削除）
2. 分析実行を試みる

**期待結果**: エラーメッセージが表示される

### 必須項目未入力で分析実行

1. プロフィールまたは案件の必須項目を空にする
2. 分析実行を試みる

**期待結果**: 入力を促すメッセージまたはボタンが無効化される

### 不正なAPIキーで分析実行

1. 無効なAPIキーを設定
2. 分析実行を試みる

**期待結果**: エラーメッセージが表示される

## 備考

- v1ではログイン機能はありません
- すべてのデータはlocalStorageに保存されます
- APIキーは暗号化されずに保存されるため、共有PCでの使用には注意が必要です
