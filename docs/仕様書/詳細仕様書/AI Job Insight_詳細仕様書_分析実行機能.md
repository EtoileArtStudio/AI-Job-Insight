# AI Job Insight 詳細設計書 - 分析実行機能

---

# 1. 機能概要

## 1.1 機能名

分析実行機能

## 1.2 機能ID

FUNC-ANALYZE-001

## 1.3 機能の目的

ユーザーが入力した案件情報およびプロフィール情報をAIに送信し、案件への応募推奨度および評価結果を取得する。

## 1.4 利用される画面

- メイン画面 分析操作領域
- メイン画面 分析結果表示領域

---

# 2. 処理の実現方式

この機能は以下の技術・方式で実現する。

- フロントエンド：React
- 言語：TypeScript
- AI呼び出し：OpenAI API / Google Gemini API
- 通信方式：HTTPS REST API
- 非同期処理：async / await
- エラーハンドリング：try / catch
- クライアントから直接AI APIを呼び出す

---

# 3. 入力データ

|項目名|型|必須|説明|
|---|---|---|---|
|AIサービス種別|string|○|使用するAIサービス（OpenAI / Gemini）|
|APIキー|string|○|AIサービスのAPIキー|
|モデル名|string|○|使用するAIモデル名（例：gpt-4, gemini-pro）|
|案件説明|string|○|ユーザーが入力した案件説明文（仕事内容、条件、報酬を含む全文）|
|案件URL|string|任意|案件URL（分析には使用しない）|
|案件メモ|string|任意|ユーザーのメモ（分析には使用しない）|
|自己紹介|string|○|プロフィールの自己紹介|
|スキル|string|○|プロフィールのスキル情報|
|実績|string|○|プロフィールの実績情報|
|得意分野|string|○|プロフィールの得意分野|
|スキル項目ラベル|string[]|任意|スキル適合度表示用の項目ラベル|

---

# 4. 出力データ

|項目名|型|説明|
|---|---|---|
|おすすめ度|number|1〜5の星評価|
|評価理由|string|AIによる案件評価理由|
|良い点|string|案件の良い点|
|注意点|string|案件の注意点|
|スキル適合度データ|object|レーダーチャート表示用データ|
|応募戦略|string|応募する場合の戦略|
|強調すべきスキル|string|強調すると良いスキル|
|提案ポイント|string|提案時のポイント|
|応答促進メッセージ|string|追加対話を促す案内文|

---

# 5. 処理の流れ

1. ユーザーが分析ボタンを押下
2. 入力データの取得
3. 入力チェック実施（必須項目の確認）
4. APIキー・モデル名の確認
5. ローディング表示開始
6. AI APIへリクエスト送信（AIサービス種別、APIキー、モデル名を含む）
7. AIレスポンス受信待機
8. レスポンス取得
9. AI応答整形処理の呼び出し
10. 整形されたデータを結果表示機能へ渡す
11. 履歴データとして保存
12. ローディング表示終了
13. 分析結果表示領域へ遷移

---

# 6. 使用するデータ

|データ名|保存場所|説明|
|---|---|---|
|APIキー|localStorage|ユーザーが設定したAIサービスのAPIキー|
|プロフィール情報|localStorage|ユーザーが入力したプロフィール情報|
|案件情報|一時保持|ユーザーが入力した案件情報|
|分析履歴|localStorage|過去の分析結果（保存後）|

# 7. AIプロンプト仕様

AIに送信するプロンプト構造を定義する。

**Note（Phase 3以降）**: プロンプトの実装は `src/domains/job/prompts/analysisPrompt.ts` の `getAnalysisPrompt()` に移管済み。以下はプロンプト内容のリファレンス定義。

## 7.1 プロンプト構造

```
あなたはクラウドソーシング案件の分析を専門とするAIアシスタントです。

以下のプロフィール情報と案件情報をもとに、この案件への応募価値を分析してください。

【プロフィール情報】
自己紹介：{自己紹介}
スキル：{スキル}
実績：{実績}
得意分野：{得意分野}

【案件情報】
案件説明：{案件説明}

【分析項目】
1. 応募おすすめ度を1〜5の星で評価してください
2. 案件の良い点を記述してください
3. 案件の注意点を記述してください
4. スキル適合度を以下の項目ごとに0〜10で評価してください：{スキル項目ラベル}
5. 応募する場合の戦略を提示してください
6. 強調すると良いスキルを記述してください
7. 提案時のポイントを記述してください

次のJSON形式で回答してください。
```

## 7.2 出力フォーマット

JSON形式

```json
{
  "score": 4,
  "goodPoints": "案件の良い点の詳細",
  "concerns": "案件の注意点の詳細",
  "skillMatch": {
    "文書作成": 8,
    "Web構築": 6,
    "プログラミング": 7,
    "仕様検討": 5,
    "イラスト作成": 2
  },
  "strategy": "応募戦略の詳細",
  "keySkills": "強調すべきスキルの詳細",
  "proposalPoints": "提案ポイントの詳細"
}
```

# 8. エラー処理

|エラー内容|対応|
|---|---|
|APIキー未設定|エラーメッセージ「APIキーが設定されていません」を表示|
|必須項目未入力|エラーメッセージ「必須項目を入力してください」を表示|
|API通信失敗|エラーメッセージ「AI APIとの通信に失敗しました」を表示|
|APIレート制限エラー|エラーメッセージ「APIの利用制限に達しました。しばらく待ってから再度お試しください」を表示|
|レスポンス解析失敗|エラーメッセージ「AIの応答を解析できませんでした」を表示|
|タイムアウト|エラーメッセージ「応答時間が長すぎます。再度お試しください」を表示|

# 10. 制約事項

- API通信はHTTPSのみ対応
- 目標応答時間：30秒以内
- APIレート制限はAIサービス側の仕様に依存
- 入力文字数制限はAIサービス側の仕様に依存
- クライアントから直接AI APIを呼び出すため、CORSエラーに注意
- スキル項目ラベルが未設定の場合はデフォルト値を使用
