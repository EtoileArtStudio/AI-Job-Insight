# AI Job Insight 詳細設計書 - AI応答サービス機能

---

# 1. 機能概要

## 1.1 機能名

AI応答サービス機能

## 1.2 機能ID

FUNC-AI-SERVICE-001

## 1.3 機能の目的

外部AIサービスとの通信を行い、分析リクエストの送信およびレスポンスの受信処理を実行する。取得したAI応答を、アプリケーション内で利用可能な形式へ整形する。

## 1.4 利用される画面

- なし（データ処理機能のため画面なし。v1ではフロントエンド実装、v2以降はバックエンド実装も追加）

---

# 2. 処理の実現方式

この機能は以下の技術・方式で実現する。

- 言語：TypeScript
- 通信方式：HTTPS REST API
- 非同期処理：async / await
- エラーハンドリング：try / catch
- v1：フロントエンド側に実装
- v2以降：フロントエンド側とバックエンド側それぞれに実装

対応AIサービス

- OpenAI API
- Google Gemini API

v2以降の追加対応

- Google Gemini API（OAuth認証によるバックエンド通信）

---

|項目名|型|必須|説明|
|---|---|---|---|
|AIサービス種別|string|○|使用するAIサービス（OpenAI / Gemini）|

|---|---|---|
|生レスポンス|string|AIから返却された生の応答テキスト|

|エラー情報|object|エラー発生時の詳細情報|

---

# 5. 処理の流れ

## 5.1 AI通信処理ブロック

1. AIサービス種別の判定
2. 該当するAI APIのエンドポイントURLを取得
3. リクエストヘッダーの構築
4. リクエストボディの構築
5. AI APIへHTTPSリクエスト送信
6. レスポンス待機
7. レスポンス受信
8. HTTPステータスコードの確認
9. エラーがある場合はエラー情報を返却
10. 正常な場合はレスポンスボディを返却

## 5.2 AI応答整形ブロック

1. 生レスポンステキストの取得
2. JSON形式への変換試行
3. パース成功時は構造化データとして返却
4. パース失敗時はテキストとして返却
5. 必要に応じてデータ構造の正規化
6. 画面表示用フォーマットへの変換
7. 履歴保存用フォーマットへの変換
8. 整形済みデータの返却

# 7. AIプロンプト仕様

この機能ではプロンプトの内容は定義せず、呼び出し元から受け取ったプロンプトをそのままAI APIへ送信する。

プロンプトの内容定義は各呼び出し元機能で行う。

## 7.1 チャットモード別システムプロンプト（2026年3月13日追加）

チャット機能において、`mode`パラメータに応じて異なるシステムプロンプトを使用する。

### 7.1.1 相談モード（consultation）

**使用場面**：案件分析ページのAI相談チャット

**システムプロンプト**：
```
あなたはクラウドソーシング案件の専門アドバイザーです。
ユーザーが案件について相談する際に、適切なアドバイスを提供してください。

重要なルール：
- ユーザーから明示的に応募文の作成を依頼された場合のみ、応募文を作成してください
- それ以外の場合は、案件の分析、マッチング度の評価、リスクの指摘、アドバイスの提供など、相談対応に徹してください
- 応募文を勝手に提案しないでください

対応内容：
- 案件内容の分析と解説
- プロフィールとのマッチング度評価
- 案件のリスクや注意点の指摘
- 応募すべきかどうかのアドバイス
- 報酬や条件の妥当性評価
- その他、案件に関する質問への回答
```

### 7.1.2 応募文作成モード（application）

**使用場面**：応募文作成ページのAIチャット

**システムプロンプト**：
```
あなたはクラウドソーシング案件への応募を支援するAIアシスタントです。
ユーザーのプロフィール情報と案件情報をもとに、効果的な応募文の作成をサポートしてください。

対応内容：
- 応募文の作成と提案
- 応募文の改善アドバイス
- アピールポイントの整理
- 文章の添削と修正提案
```

### 7.1.3 実装詳細

**Phase 3（共通基盤リファクタリング）により実装場所が変更されている。**

- `ChatRequest`インターフェースに`mode?: 'application' | 'consultation'`パラメータを追加
- システムプロンプト生成関数は `src/domains/job/prompts/chatPrompts.ts` に分離済み：
  - `getConsultationSystemPrompt()`：相談モード用
  - `getApplicationSystemPrompt()`：応募文作成モード用
- チャット実行は `src/domains/job/services/jobChatService.ts` の `chatWithAI()` が担う
- AI通信の実体は `src/core/ai/client.ts` の `callAIChat()` が担う（`chatWithOpenAI()` / `chatWithGemini()` は内部プライベート関数）

# 8. エラー処理

|エラー内容|対応|
|---|---|
|APIキー未設定|エラーコード「API_KEY_MISSING」を返却|
|ネットワークエラー|エラーコード「NETWORK_ERROR」を返却|
|HTTP 401エラー|エラーコード「UNAUTHORIZED」を返却（APIキー無効）|
|HTTP 429エラー|エラーコード「RATE_LIMIT」を返却（レート制限）|
|HTTP 500エラー|エラーコード「SERVER_ERROR」を返却（サーバーエラー）|
|タイムアウト|エラーコード「TIMEOUT」を返却|
|レスポンスパースエラー|エラーコード「PARSE_ERROR」を返却|
|不明なエラー|エラーコード「UNKNOWN_ERROR」を返却|

# 10. 制約事項

- 通信はHTTPSのみ対応
- タイムアウト時間：30秒
- リトライ回数：なし（エラー時はユーザーが再実行）
- レート制限はAIサービス側の仕様に依存
- v1ではCORS制約に注意が必要
- v2以降のOAuth認証はバックエンド側で処理

# 11. デモモード対応

## 11.1 概要

APIキーまたはモデル名が未設定の場合、デモモードが自動的に有効化され、実際のAI通信を行わずに固定のデモデータを返す。

## 11.2 デモモード判定

```typescript
function isDemoMode(): boolean {
  const config = getStorageItem<ApiKeyConfig>(STORAGE_KEYS.API_KEY_CONFIG);
  return !config || !config.apiKey || !config.modelName;
}
```

## 11.3 デモデータ定義

- **demoProfile**: デモ用プロフィールデータ
- **demoJob**: デモ用案件データ
- **demoAnalysisResult**: デモ用分析結果
- **demoAIResponses**: デモ用AIチャット応答（パターン別）
- **demoGeneratedProfileText**: デモ用プロフィール文
- **demoApplicationSuggestions**: デモ用応募文（初回・改善）

## 11.4 デモ応答の特徴

- 実際のAI通信を行わない
- 疑似的な遅延（1～1.5秒）を追加してリアルな体験を提供
- キーワードに応じたパターンマッチング応答
- ユーザーがアプリケーションの機能を体験できる

## 11.5 デモモード時の制限

- 実際のAI応答とは異なる固定データが返される
- カスタマイズされた分析結果は提供されない
- 通常モードに移行するにはAPIキーの設定が必要

# 13. 備考

## OpenAI API仕様

- エンドポイント：https://api.openai.com/v1/chat/completions
- 認証方式：Bearer Token
- リクエスト形式：JSON
- レスポンス形式：JSON

## Google Gemini API仕様（v1）

- エンドポイント：https://generativelanguage.googleapis.com/v1/models/{model}:generateContent
- 認証方式：APIキーパラメータ
- リクエスト形式：JSON
- レスポンス形式：JSON

## Google Gemini API仕様（v2以降 OAuth）

- OAuth認証経由での利用
- バックエンドサーバー経由で通信
- ※本機能はv2以降で実装予定

- API_KEY_MISSING：APIキー未設定

- UNAUTHORIZED：認証エラー
- RATE_LIMIT：レート制限エラー
- SERVER_ERROR：サーバーエラー
- TIMEOUT：タイムアウトエラー
- PARSE_ERROR：パースエラー
- UNKNOWN_ERROR：不明なエラー
